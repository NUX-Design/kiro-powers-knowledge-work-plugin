# SOX Testing

Generate sample selections, testing workpapers, control matrices, and deficiency assessment structures for SOX 404 work.

This workflow assists with control testing and should be reviewed by qualified professionals before audit use.

## Typical Inputs

- control area such as revenue, procure-to-pay, payroll, financial close, treasury, fixed assets, inventory, ITGC, entity-level, or journal entries
- testing period
- control frequency
- population details if available

## Workflow

### 1. Identify Controls

Build a control matrix:

```markdown
| Control # | Control Description | Type | Frequency | Key/Non-Key | Risk | Assertion |
|-----------|-------------------|------|-----------|-------------|------|-----------|
| [ID]      | [Description]     | Manual/Automated/IT-Dependent | Daily/Weekly/Monthly/Quarterly/Annual | Key | High/Medium/Low | [CEAVOP] |
```

### 2. Determine Sample Size

Adjust sample size based on:

- control frequency
- risk level
- prior-year results
- auditor reliance

### 3. Generate Sample Selection

Use random, systematic, or targeted selection as appropriate and document methodology explicitly.

### 4. Create Testing Workpaper

Include:

- test objective
- step-by-step procedures
- expected evidence
- per-sample results table
- exception log
- conclusion section

### 5. Provide Control-Specific Templates

Pre-populate test steps for areas such as:

- revenue recognition
- procure to pay
- financial close
- IT general controls

### 6. Classify Exceptions

Map exceptions to:

- effective
- effective with exceptions
- deficiency
- significant deficiency
- material weakness

## Output

Provide:

- control matrix
- sample selection table
- testing workpaper template
- results template
- deficiency evaluation guidance
- suggested remediation actions
