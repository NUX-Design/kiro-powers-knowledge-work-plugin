# Scientific Problem Selection

Use a structured conversational framework to choose research problems, evaluate ideas, troubleshoot stuck work, and make strategic scientific decisions.

## Entry Points

Start from one of three modes:

1. pitch a new idea
2. troubleshoot a current project problem
3. ask a strategic question

## Core Principle

Problem choice matters more than execution quality alone. Spend meaningful effort choosing the right problem before committing long cycles of work.

## Workflow Modules

The source framework groups work into nine conceptual modules:

- intuition and idea refinement
- risk assessment
- optimization and success metrics
- parameter strategy
- decision tree navigation
- adversity planning
- problem inversion
- integration and synthesis
- end-to-end orchestration

## How To Apply

### New Idea

- ask for the short version first
- summarize understanding briefly
- then gather detail on approach, significance, and risk

### Stuck Project

- ask for the short version of the problem
- summarize the core failure mode
- gather project goal, what went wrong, and current fixes under consideration

### Strategic Question

- ask for the short version of the question
- confirm the crux
- expand into future-planning or current-project strategy as appropriate

## Output Expectations

Produce concrete artifacts such as:

- risk matrices
- decision trees
- parameter strategies
- adversity plans
- integrated project narratives

## Best Practices

- ask clarifying questions when confidence is low
- make risks explicit instead of avoiding them
- identify which constraints should stay fixed and which should remain flexible
- integrate literature evidence when evaluating generality, feasibility, or impact
