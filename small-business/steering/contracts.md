# Contract Review

Use this workflow when a small-business owner needs a plain-English read on a contract before signing or negotiating.

This steering file is derived primarily from the source plugin’s `contract-review` and wrapper-style `review-contract` behavior.

## Inputs

Accept contracts from:

- uploaded files
- pasted text
- Gmail attachments
- DocuSign envelopes

Read the full document, including exhibits and schedules, before analyzing.

## Core Review Areas

Review from the perspective of an owner without in-house counsel. Prioritize:

1. Payment terms and cash-flow impact
2. Liability and indemnification
3. Termination and exit
4. Intellectual property
5. Scope and change management
6. Non-compete and exclusivity
7. Confidentiality and data handling
8. Operational boilerplate with real business impact

## Output Structure

Provide:

- `Red flags`: push back before signing
- `Yellow flags`: negotiate if possible
- `Key terms to note`: awareness items
- `Contract summary`: plain-English description of the deal
- `Negotiation playbook`: what to ask for and how to frame it

For each flagged clause:

- quote the actual language
- explain the issue in plain English
- compare to a reasonable market norm
- propose better or safer wording when useful

## Redline Export

After the review summary, offer to create a redlined document with suggested changes clearly marked.

## Guardrails

- Never frame the result as legal advice.
- Do not invent clause text.
- Do not flag every clause mechanically; focus on signal.
- Adjust recommendations to the negotiation power dynamic.
- Never send the redline externally without explicit user approval.
