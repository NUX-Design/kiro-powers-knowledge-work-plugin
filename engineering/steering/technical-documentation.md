# Technical Documentation

Write clear, maintainable technical documentation for different audiences and purposes.

## Document Types

### README

- What this is and why it exists
- Quick start in under five minutes
- Configuration and usage
- Contributing guidance

### API Documentation

- Endpoint reference with request and response examples
- Authentication and error codes
- Rate limits and pagination
- SDK examples

### Runbook

- When to use it
- Prerequisites and access required
- Step-by-step procedure
- Rollback steps
- Escalation path

### Architecture Doc

- Context and goals
- High-level design with diagrams
- Key decisions and trade-offs
- Data flow and integration points

### Onboarding Guide

- Environment setup
- Key systems and relationships
- Common tasks and walkthroughs
- Ownership and escalation routes

## Principles

1. Write for the reader and their immediate task.
2. Lead with the most useful information.
3. Prefer concrete examples, commands, and screenshots over abstraction.
4. Keep documentation current enough to trust operationally.
5. Link to canonical sources instead of duplicating content.
