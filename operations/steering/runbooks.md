# Operational Runbooks

Create or update a runbook for a recurring operational task or procedure.

## Core Sections

Include:

- purpose
- prerequisites
- exact procedure
- expected results per step
- failure handling per step
- verification checklist
- troubleshooting table
- rollback procedure
- escalation contacts
- run history

## Best Practices

- make steps painfully specific
- include exact commands or actions where possible
- document failure modes at the point they can occur
- include verification and rollback, not just the happy path
- test the runbook with someone unfamiliar to find ambiguity

## Connected-Source Enhancements

- search a knowledge base for existing runbooks before starting from scratch
- link related incidents or change records through ITSM when available
