# Process Documentation

Turn a business process into a usable SOP with flow, ownership, exceptions, and metrics.

## Workflow

1. capture the purpose and scope
2. identify the owner and review cadence
3. map the process flow
4. define step-by-step instructions
5. clarify roles with a RACI
6. document exceptions and edge cases
7. identify operational metrics and related documents

## Output Structure

Produce:

- purpose
- scope
- RACI matrix
- process flow
- detailed steps
- exceptions and edge cases
- success metrics
- related documents or policies

## Best Practices

- start from how the process actually works today
- capture exceptions, not just the happy path
- include names or roles where ownership is unclear
- update existing documentation rather than duplicating it when a knowledge base already has relevant material
