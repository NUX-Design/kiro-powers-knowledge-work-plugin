# Change Requests

Create structured change requests with impact analysis, implementation planning, communication plans, and rollback readiness.

## Framework

Use an assess, plan, execute, sustain structure.

### Assess

- what is changing
- who is affected
- how significant it is
- what resistance or disruption may occur

### Plan

- communication plan
- training plan
- support plan
- implementation timeline and dependencies

### Execute

- announce the change and explain the why
- enable adoption
- monitor rollout and resistance

### Sustain

- measure adoption and effectiveness
- reinforce new behaviors
- document lessons learned

## Output

Produce:

- change description
- business justification
- impact analysis table
- risk assessment table
- implementation plan
- communication plan
- rollback plan
- approvals required

## Connected-Source Enhancements

- use ITSM systems for change records and approval workflows
- use project trackers for linked implementation tasks
- use chat tools for stakeholder notification drafts
